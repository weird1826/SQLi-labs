# SQLi-Playground
---

This is a Nest-JS based web application which is intended to provide a vulnerable environment to practice for SQL injections. The way it works that there are a bunch of API routes in this application, which are named as levels, so level1, level2, level3, and so on...

Each level has their own "set" of vulnerabilities - or let's say that we can practice more advanced payloads and techniques as we walk up to the levels. Nothing is locked or there are no constraints to these levels. You can start at any level you prefer.

Do not think that the vulnerabilities or the expected payloads are hardcoded into the backend; they are not. The application is genuinely vulnerable with logic issues and unsafe practices and it is genuinely fun to experiment with this!

If you find anything much more than we are providing, or generally have anything to say about this, please reach out!

I will be uploading guide and writeups for this application shortly.
